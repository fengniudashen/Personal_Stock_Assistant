### 🚀 核心决策总结 (Executive Summary)

*   **模型架构**：**MobileNetV3-Small** (ImageNet 预训练)。
    *   *理由*：iOS Neural Engine (ANE) 亲儿子，推理 <10ms，开发成本最低，社区支持最好。
*   **音频参数**：**44.1kHz, Mel-128, fmax=16000Hz**。
    *   *理由*：必须保留高频以检测“漏气 (Breathy)”，必须保留低频分辨率以检测“真声 (Chest)”。
*   **训练策略**：**MVP 离线预处理** -> **Scale 阶段 GPU 在线计算 + DDP**。
*   **部署策略**：**iOS 端计算频谱 (vDSP)** -> **CoreML (FP16)**。

### 📅 阶段一：MVP 快速验证 (Local RTX 5060)
**目标**：跑通 "录音 -> 频谱 -> 模型 -> 结果" 的全链路，验证 5 分类可行性。

#### 1. 模型定义 (PyTorch)
直接修改 `torchvision` 的官方模型，最稳。

```python
import torch
import torch.nn as nn
from torchvision.models import mobilenet_v3_small, MobileNet_V3_Small_Weights

class VibeSingModel(nn.Module):
    def __init__(self, num_classes=5):
        super().__init__()
        # 1. 加载预训练权重 (迁移学习关键)
        weights = MobileNet_V3_Small_Weights.DEFAULT
        self.backbone = mobilenet_v3_small(weights=weights)
        
        # 2. 修改第一层卷积：适配 1通道 Mel谱
        # 技巧：将 RGB 3通道权重的平均值赋值给新的 1通道卷积，保留纹理提取能力
        original_first_layer = self.backbone.features[0][0]
        new_first_layer = nn.Conv2d(1, 16, kernel_size=3, stride=2, padding=1, bias=False)
        with torch.no_grad():
            new_first_layer.weight[:] = original_first_layer.weight.mean(dim=1, keepdim=True)
        self.backbone.features[0][0] = new_first_layer
        
        # 3. 修改分类头 (Classifier)
        in_features = self.backbone.classifier[3].in_features
        self.backbone.classifier[3] = nn.Linear(in_features, num_classes)

    def forward(self, x):
        return self.backbone(x)
```

#### 2. 音频特征参数 (固定标准)
为了捕捉“漏气”和“挤卡”，参数不能太粗糙。

*   **SR**: 44100 Hz (不要降采样，保留高频细节)
*   **n_fft**: 2048 (约 46ms，保证频率分辨率)
*   **hop_length**: 512 (约 11.6ms，保证时间分辨率捕捉 Jitter/Shimmer)
*   **n_mels**: 128
*   **f_min**: 50 Hz
*   **f_max**: **16000 Hz** (关键！8k 不足以完美捕捉气声)
*   **Input Shape**: `[Batch, 1, 128, 258]` (3秒音频)

#### 3. MVP 训练策略
*   **数据处理**：写脚本用 `librosa` 预先生成 `.npy` 文件存硬盘。
    *   *原因*：RTX 5060 很快，但 Python 的 `librosa` 是 CPU 单线程，会卡住 GPU。预处理能解决此瓶颈。
*   **Augmentation**：仅使用 **SpecAugment** (TimeMasking, FrequencyMasking)。

### 📅 阶段二：Scale-Up 规模化训练 (Cloud 2x RTX 4090)
**目标**：利用 10k+ 数据提升精度，解决过拟合。

#### 1. 解决 I/O 瓶颈 (GPU Transforms)
数据量大时，硬盘读取和 CPU 预处理会太慢。改用 `torchaudio` 在 GPU 上做变换。

```python
import torchaudio.transforms as T

class GPUTransform(nn.Module):
    def __init__(self):
        super().__init__()
        # 这一层放在 GPU 上跑
        self.mel_transform = T.MelSpectrogram(
            sample_rate=44100, n_fft=2048, hop_length=512, 
            n_mels=128, f_min=50, f_max=16000, normalized=True
        )
        self.amplitude_to_db = T.AmplitudeToDB()

    def forward(self, waveform):
        # waveform: [Batch, Time] -> GPU Tensor
        spec = self.mel_transform(waveform)
        spec = self.amplitude_to_db(spec)
        return spec # [Batch, 1, 128, 258]
```

#### 2. DDP 训练配置 (PyTorch Lightning)
利用双卡 48GB 显存优势。

*   **Batch Size**: 单卡 256 -> 全局 **512**。
*   **Precision**: `16-mixed` (混合精度)。4090 在 FP16 下速度飞快。
*   **SyncBatchNorm**: **True** (多卡训练必须开启，否则 BN 层统计不准)。
*   **Learning Rate**: 使用 **OneCycleLR** 策略。
    *   它会自动处理 Warm-up（热身）和 Decay（衰减）。
    *   `max_lr` 设为 `1e-3` * (Global_Batch / 256)。

### 📅 阶段三：iOS 部署与落地 (The "Last Mile")
**目标**：<200ms 延迟，低功耗。

#### 1. CoreML 转换 (Python)
不要把 STFT (频谱变换) 放在模型里，CoreML 对此支持不好且慢。**模型只接受频谱图输入。**

```python
import coremltools as ct

# 1. 导出 Trace
model.eval()
example_input = torch.randn(1, 1, 128, 258) 
traced_model = torch.jit.trace(model, example_input)

# 2. 转换
mlmodel = ct.convert(
    traced_model,
    inputs=[ct.TensorType(name="mel_input", shape=(1, 1, 128, 258))], # 固定 Shape
    minimum_deployment_target=ct.target.iOS17, # 利用最新 NPU 特性
    compute_units=ct.ComputeUnit.ALL
)

# 3. FP16 量化 (推荐)
# 相比 Int8，FP16 在 ANE 上通常更快，且精度无损
mlmodel = ct.models.neural_network.quantization_utils.quantize_weights(
    mlmodel, nbits=16
)
mlmodel.save("VibeSing_MobileNetV3.mlpackage")
```

#### 2. iOS 端音频处理 (Swift)
这是最容易卡壳的地方。你需要用 Swift 的 `Accelerate` 框架 (vDSP) 来计算 Mel 频谱，必须与 Python 端 `librosa` 的参数**严格对齐**。

*   **推荐库**：不要自己手写 vDSP，使用 **AudioKit** 或 **Piano (Github)** 等现成库来生成频谱图。
*   **流程**：
    1.  `AVAudioEngine` 录制音频 (Buffer)。
    2.  转为 `Float` 数组。
    3.  计算 STFT -> Mel Filter Bank -> Log (dB)。
    4.  构建 `MLMultiArray`。
    5.  `model.prediction(input: ...)`。

### ⚠️ 避坑指南 (针对 Solo Dev)

1.  **不要降采样到 16k/22k**：虽然省算力，但你会丢失“漏气”的高频特征，导致模型在这个类别上瞎猜。
2.  **不要在 Swift 里做归一化**：在导出 CoreML 时，使用 `preprocessing_args` 把 `mean` 和 `std` 烧录进模型。Swift 端只传原始 dB 值。
3.  **不要追求 Transformer (AST)**：在移动端，CNN (MobileNet) 依然是能效比之王。Transformer 会导致手机发热，用户体验变差。
4.  **数据不平衡**：如果“挤卡”样本很少，训练时使用 `WeightedRandomSampler`，否则模型会倾向于预测“真声”。
