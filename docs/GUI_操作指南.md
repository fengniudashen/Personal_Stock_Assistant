## 5. 音频提取 (Step 1)

将视频文件转换为标准化音频格式。

### 操作
1. 设置 **输入目录** (视频文件所在)
2. 设置 **输出目录** (音频输出位置)
3. 确认 **采样率** (默认 44100Hz，声乐分析推荐)
4. 点击 **🎵 提取音频**

### 技术说明
- 输出格式: WAV (16bit, Mono)
- 自动跳过已转换的文件
- 支持格式: mp4, webm, mp3, m4a, flac 等

---

## 6. 人声分离 (Step 2)

使用 AI 模型将人声从伴奏中分离。

### 操作
1. 设置 **输入目录** (原始音频)
2. 设置 **输出目录** (干净人声)
3. 选择 **分离方法**:
   - `demucs (htdemucs)` - Meta 开源，效果好 ⭐推荐
   - `UVR5` - Ultimate Vocal Remover
   - `MDX-Net` - 快速但质量稍低
4. 点击 **🔊 开始分离**

### 注意事项
- 首次运行会自动下载模型 (~100MB)
- GPU 可用时自动加速 (CUDA)
- 处理 1 分钟音频约需 10-30 秒

## 7. 智能切片 (Step 3)

将长音频切分为 2-6 秒的短片段。

### 切片方法
| 方法 | 说明 |
|------|------|
| `hybrid` | VAD + 能量检测混合 ⭐推荐 |
| `vad` | 纯 WebRTC VAD |
| `energy` | 基于能量阈值 |

### 关键参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| 目标时长 | 4.0s | 理想切片长度 |
| 最小时长 | 2.5s | 短于此的被丢弃 |
| 最大时长 | 6.0s | 超过此的被再次切割 |

## 8. ASR 识别 (Step 4)

使用 Whisper 模型进行语音识别，提取关键词用于弱标签。

### 模型选择
| 模型 | 大小 | 速度 | 准确度 |
|------|------|------|--------|
| `tiny` | 39M | ⚡ 极快 | ⭐ |
| `base` | 74M | ⚡ 快 | ⭐⭐ |
| `small` | 244M | 中等 | ⭐⭐⭐ |
| `medium` | 769M | 较慢 | ⭐⭐⭐⭐ |
| `large-v3` | 1.5G | 最慢 | ⭐⭐⭐⭐⭐ |

### 关键词映射说明
系统会根据 `config_advanced.yaml` 中的关键词映射，将 ASR 识别到的文本自动匹配到标签：

**主标签关键词示例：**
- `StrongMix`: "强混", "strong mix", "胸混"
- `Strained`: "挤", "挤卡", "喉部紧张", "卡嗓"
- `Breathy`: "漏气", "气声", "闭合不足"

**辅标签关键词示例：**
- `Pharyngeal`: "咽音", "咽腔", "挂咽壁"
- `Twang`: "收窄", "鼻腔收窄", "哭腔"

## 9. 特征提取 (Step 5)

提取声学特征，供启发式规则和模型使用。

### 提取的特征
| 特征 | 说明 | 用途 |
|------|------|------|
| Mel 频谱图 | 128维频谱 | 模型输入 |
| H1-H2 | 谐波差 | 区分气声/挤卡 |
| HNR | 谐波噪声比 | 声音清晰度 |
| Jitter | 音高微扰 | 声音稳定性 |
| Shimmer | 振幅微扰 | 声音稳定性 |
| Spectral Tilt | 频谱倾斜 | 胸声/头声倾向 |
| F0 | 基频 | 音高范围判定 |

### 启发式特征开关
可通过复选框独立控制每种特征的提取，减少不需要的计算。

## 10. 弱标签融合 (Step 6)

融合多路信号生成v2标签(主标签+辅标签+质量旗标)。

### 融合源
| 信号源 | 默认权重 | 说明 |
|--------|---------|------|
| ASR 关键词 | 0.4 | 从语音识别中提取 |
| 启发式规则 | 0.2 | 基于声学特征的先验规则 |
| 模型预测 | 0.3 | 教师模型推理 |
| 近邻传播 | 0.1 | 基于嵌入相似度 |

### v2 新增能力
- **辅标签自动检测**：启发式分类器同时输出 Pharyngeal、Twang、HighRange 等辅标签概率
- **质量旗标自动检测**：自动标记 LowSNR、Clipping、Unstable 等质量问题
- **输出格式**：每条结果包含 `suggested_label`(主) + `secondary_tags`(辅) + `quality_flags`(质量)

### 权重调节指南
- 有大量教学视频 (ASR 丰富) → 提高 ASR 权重
- 训练了高质量教师模型 → 提高模型权重
- 冷启动 (无模型) → ASR=0.5, 启发式=0.4, 近邻=0.1

## 11. 嵌入计算 (Step 7)

计算音频嵌入向量，用于聚类、去重和近邻传播。

### 可用模型
| 模型 | 维度 | 说明 |
|------|------|------|
| `openl3` | 512 | 通用音频嵌入 ⭐推荐 |
| `clap` | 512 | 文本-音频对齐嵌入 |
| `hubert` | 768 | 语音自监督模型 |

## 12. 聚类去重 (Step 8)

对嵌入向量进行聚类，发现自然分组并去除重复。

### 参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| 聚类方法 | `hdbscan` | 自适应密度聚类 |
| 最小簇大小 | 3 | 簇至少需要的样本数 |
| 去重阈值 | 0.95 | 余弦相似度 >95% 视为重复 |

### 输出
- 每个切片分配 `cluster_id`
- 重复样本标记 (可选移除)
- 簇内样本分布统计

## 13. 主动学习 (Step 9)

智能选择最有价值的样本进行人工标注。

### 策略
| 策略 | 说明 |
|------|------|
| `uncertainty` | 选择模型最不确定的样本 |
| `diversity` | 选择多样性最大的样本 |
| `uncertainty_diversity` | 综合两者 ⭐推荐 |

### 参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| 批次大小 | 50 | 每轮选择的样本数 |
| 不确定性权重 | 0.7 | uncertainty vs diversity 的比例 |

### 输出
生成 `data/annotation_queue.json` — 按优先级排序的待标注列表。

## 14. 🔍 人工审核 (Step 10)

**核心原则:** 所有音频切片在进入最终训练数据集前，必须经过人工逐条审核确认。

### 为什么需要人工审核？

自动标签（弱监督融合结果）虽然效率高，但不可避免存在误判。通过人工审核可以：
- 纠正自动标签的错误分类
- 剔除质量不达标的切片
- 修正辅标签和质量旗标
- 确保训练数据集的高质量

### 界面布局

| 区域 | 说明 |
|------|------|
| 工具栏 | 加载切片、状态筛选、标签筛选、批量操作按钮 |
| 统计栏 | 实时显示总计/待审/通过/拒绝/进度 |
| 切片列表（左） | 所有切片的 ID、主标签、置信度、辅标签、审核状态 |
| 详情面板（右） | 文件信息、音频播放、自动标签展示、标签修改、审核按钮 |

### 操作流程

1. 点击 **🔄 加载切片** 加载所有待审核切片
2. 在左侧列表选择一条切片
3. 右侧详情面板显示自动标签和音频信息
4. 点击 **▶ 播放** 试听音频
5. 如需修改标签：在编辑区修改主标签/辅标签/质量旗标
6. 点击 **✅ 通过** 或 **❌ 拒绝**
7. 自动前进到下一条

### 快捷键

| 快捷键 | 功能 |
|--------|------|
| `A` | 通过当前切片 |
| `R` | 拒绝当前切片 |
| `S` | 跳过（不改变状态） |
| `Space` | 播放/停止音频 |
| `←` / `↑` | 上一条 |
| `→` / `↓` | 下一条 |

> ⚠️ 快捷键仅在审核面板激活且焦点不在输入框时生效

### 批量操作

| 操作 | 说明 |
|------|------|
| **⚡ 高置信度自动通过** | 将置信度 ≥ 0.92 且无冲突的切片自动标记为通过 |
| **🗑️ 自动拒绝无效** | 将标签为 Invalid 或有多项严重质量问题的切片自动拒绝 |

### 筛选功能

- **状态筛选:** 全部 / ⏳待审核 / ✅已通过 / ❌已拒绝
- **标签筛选:** 按主标签类型筛选

### 审核结果存储
- 文件: `data/review_results.json`
- 格式: `{ "clip_id": { "review_status", "final_primary_label", "final_secondary_tags", "final_quality_flags", "reviewer_notes", "reviewed_at" } }`

### 与导出的关系
- 导出面板新增 **"✅ 仅导出审核通过的切片"** 选项（默认勾选）
- 勾选后，只有审核状态为 `approved` 的切片才会被导出到训练集
- 命令行: `python scripts/export_dataset.py --only-approved`

## 16. Label Studio 集成

### 生成标注任务
1. 确认切片目录和弱标签目录
2. 设置音频 Base URL (FastAPI 服务地址)
3. 点击 **📋 生成任务 JSON**
4. 输出文件用于导入 Label Studio

### 导入到 Label Studio
1. 确保 Label Studio 已启动 (`http://localhost:8080`)
2. 填入 **LS 地址** 和 **API Key**
3. 点击 **📤 导入到 Label Studio**

### 标注界面说明
Label Studio 使用 `label_config.xml` 配置了三步标注流程：

1. **📌 第一步 - 主标签** (单选必填)
   - 1/2/3/4/5/6/7/0/X 快捷键
   - 9个互斥选项 (StrongMix → Invalid)

2. **🎨 第二步 - 辅标签** (多选可选)
   - Pharyngeal, Twang 核心共鸣技术
   - HighRange, SuperHighRange 音高标记
   - Vibrato, Glissando, Runs 技巧标记
   - Demo_Correct, Demo_Error 教学标记

3. **🚩 第三步 - 质量旗标** (多选可选)
   - LowSNR, HighReverb, Clipping 等

## 17. 模型训练

### VibeSingMultiTaskModel 架构
```
Mel (1, 128, T)
    ↓
CNN Encoder (4× ConvBlock → AdaptivePool → FC)
    ↓
Feature Vector (256-dim)
    ├── Primary Head → 9类 softmax (发声机制)
    ├── Secondary Head → 10类 sigmoid (共鸣色彩)
    └── Quality Head → 6类 sigmoid (质量旗标)
```

### 训练参数
| 参数 | 推荐值 | 说明 |
|------|--------|------|
| Epochs | 20-50 | 数据量大可增加 |
| Batch Size | 32 | 显存不足可减小 |
| 学习率 | 0.001 | Adam 优化器 |
| 主标签权重 | 0.7 | 最重要的分类任务 |
| 辅标签权重 | 0.2 | 共鸣技术检测 |
| 质量权重 | 0.1 | 质量问题检测 |

### 类别权重
对少样本类别设置更高权重（已在配置中预设）：
- Breathy: 1.5, Strained: 1.5 (少见但重要)
- Chest: 0.5, Head: 0.5 (相对常见)

## 19. 标签体系 v2.0 说明

### 设计原则
**机制与色彩正交** — 主标签描述"声带如何振动"，辅标签描述"共鸣腔如何调整"。两者独立，可自由组合。

### 主标签 (Primary Labels) — 发声机制，互斥单选

| 标签 | 热键 | 描述 | 声学特征 |
|------|------|------|----------|
| StrongMix | 1 | 强混声 | HNR高, 闭合强, 低中高频均衡 |
| LightMix | 2 | 弱混声 | HNR中高, 低频较弱 |
| Falsetto | 3 | 假声 | 谐波稀疏, RMS低, ZCR高 |
| Chest | 4 | 胸声 | 频谱倾斜陡, F0<C4 |
| Head | 5 | 头声 | 频谱倾斜缓, 头腔共鸣 |
| Breathy | 6 | 漏气 | H1-H2>8dB, HNR低 |
| Strained | 7 | 挤卡 | 1-3kHz集中, Jitter高 |
| Neutral | 0 | 中性 | 无明显特征 |
| Invalid | X | 无效 | 非歌唱/噪音 |

### 辅标签 (Secondary Tags) — 共鸣色彩，可叠加多选

| 标签 | 描述 | 自动检测方式 |
|------|------|-------------|
| Pharyngeal | 咽音 | H1-H2低 + HNR高 + Jitter低 |
| Twang | 收窄 | 频谱倾斜缓 + HNR中高 |
| HighRange | 高音区C5+ | F0 ≥ 523Hz |
| SuperHighRange | 超高音C6+ | F0 ≥ 1047Hz |
| VowelMod | 元音修正 | 人工标注 |
| Vibrato | 颤音 | F0标准差 15-80Hz |
| Glissando | 滑音 | 人工标注 |
| Runs | 转音 | 人工标注 |
| Demo_Correct | 正确示范 | ASR关键词 |
| Demo_Error | 错误示范 | ASR关键词 |

### 质量旗标 (Quality Flags)

| 旗标 | 描述 | 自动检测 |
|------|------|---------|
| LowSNR | 信噪比低 | HNR < 5dB |
| HighReverb | 混响重 | 人工/频谱分析 |
| Clipping | 削波失真 | 峰值 >1% |
| Unstable | 音高不稳 | Jitter>0.04 + Shimmer>0.15 |
| MultiVoice | 多人声 | 人工标注 |
| Distortion | 音色失真 | 人工标注 |

### 关键区分规则

**Strained(挤卡) vs Pharyngeal(咽音):**
- 挤卡 = 不稳定、爆破感、颈部紧张、想咳嗽
- 咽音 = 稳定、明亮穿透、不紧张、可持续3秒+

**StrongMix vs LightMix:**
- 强混 = 同音高下可提升响度、低频厚实
- 弱混 = 响度提不上去但连接顺滑、低频较薄

**LightMix vs Falsetto:**
- 弱混 = 有支撑、谐波连续、尝试加力可以
- 假声 = 完全轻飘、谐波稀疏、加力就会破

## 20. 常见问题

### Q: GUI 启动时报 tkinter 找不到？
```
A: tkinter 是 Python 标准库。如果缺失，请重新安装 Python 并勾选 tcl/tk 选项。
   Windows 上默认安装包含 tkinter。
```

### Q: 运行某步骤时报 "ImportError"？
```
A: 确保虚拟环境已激活: .venv\Scripts\activate
   然后安装依赖: pip install -r requirements.txt
```

### Q: 后台任务如何取消？
```
A: 当前版本不支持中途取消。可以关闭 GUI 终止所有任务。
   后续版本将添加取消功能。
```

### Q: 如何查看标注数据？
```
A: 三种方式:
   1. 仪表盘的统计卡片
   2. API 服务: http://localhost:8000/api/stats
   3. 直接查看 SQLite 数据库: data/vibesing.db
```

### Q: 如何自定义标签体系？
```
A: 修改 config_advanced.yaml 中的 labels 部分：
   - primary: 主标签列表
   - secondary: 辅标签列表
   - quality_flags: 质量旗标列表
   修改后需同步更新 label_config.xml 中的 Label Studio 界面配置。
```

### Q: 推荐的工作流程？
```
典型流程:
1. 数据下载 (Step 0) — 收集教学视频
2. 音频提取 (Step 1) — 视频转音频
3. 人声分离 (Step 2) — 去除伴奏
4. 智能切片 (Step 3) — 分割为小段
5. ASR识别 (Step 4) — 提取文本关键词
6. 特征提取 (Step 5) — 声学特征
7. 弱标签融合 (Step 6) — 多路融合
8. 嵌入计算 (Step 7) — 音频向量
9. 聚类去重 (Step 8) — 分组去重
10. 主动学习 (Step 9) — 选样本
11. Label Studio 标注 — 人工校验
12. 数据导出 — 训练集
13. 模型训练 — VibeSingMultiTaskModel
14. 重复 6-13 直到精度满意
```
